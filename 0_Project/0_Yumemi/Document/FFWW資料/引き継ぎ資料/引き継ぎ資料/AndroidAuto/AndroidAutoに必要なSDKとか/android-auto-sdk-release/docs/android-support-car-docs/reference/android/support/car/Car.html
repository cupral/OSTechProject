Class Overview

Top-level car API that provides access to all car services and data available in the platform.

Use one of the createCar methods to create a new instance of the Car api. The CarConnectionCallback will respond with an onConnected(Car) or onDisconnected(Car) message. Nothing can be done with the car until onConnected is called. When the car disconnects then reconnects you may still use the Car object but any manages retried from it should be considered invalid and will need to be retrieved.

Once connected, getCarManager(String) or getCarManager(Class) can be used to retrieve a manager. This is patterned after how one would retrieve a service from getSystemService(String) or getSystemService(Class). Once again if the car is disconnected you'll want to get new versions of these managers.

public void disconnect ()

Disconnect from Car Service. Can be called while disconnected. After disconnect is called, all Car*Managers from this instance become invalid, and getCarManager(String) returns a different instance if connected again.

public Object getCarManager (String serviceName)

Get a car-specific manager. This is modeled after getSystemService(String). The returned Object should be type cast to the desired manager. For example, to get the sensor service, use the following: 

CarSensorManager sensorManager =
     (CarSensorManager) car.getCarManager(Car.SENSOR_SERVICE);

Parameters
serviceName String: Name of service to create, for example SENSOR_SERVICE.
Returns
Object The requested service manager or null if the service is not available.
Throws
CarNotConnectedException
