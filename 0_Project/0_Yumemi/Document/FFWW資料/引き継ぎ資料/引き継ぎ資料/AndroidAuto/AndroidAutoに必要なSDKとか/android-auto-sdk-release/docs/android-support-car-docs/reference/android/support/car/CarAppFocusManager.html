Class Overview

Enables applications to set and listen for the current application focus (such as active navigation). Typically, only one such application should be running at a time. When another application gets ownership of a given APP_FOCUS_TYPE_*, the old app should stop using the feature represented by the focus type.

public static final int APP_FOCUS_TYPE_NAVIGATION

Represents navigation focus.

When a program loses navigation focus they should no longer send navigation data to the instrument cluster via the CarNavigationStatusManager. Furthermore they should stop sending audio updates and any notifications regarding navigation. Essentially, these apps should stop all navigation activities as this means another app is navigating.

public abstract int requestAppFocus (int appType, CarAppFocusManager.OnAppFocusOwnershipCallback ownershipCallback)

Request application focus.

By requesting this, the app gains the focus for this appType

This call is asynchronous, focus may not be granted immediately. onAppFocusOwnershipGranted(CarAppFocusManager, int) will be sent to the app when focus is granted.

onAppFocusOwnershipLost(CarAppFocusManager, int) will be sent to the app that currently holds focus. The foreground app will have higher priority; other apps cannot set the same focus while owner is in foreground.

The callback provided here is the identifier for the focus. Apps need to pass it into other app focus methods such as isOwningFocus(int, OnAppFocusOwnershipCallback) or abandonAppFocus(OnAppFocusOwnershipCallback).

Parameters
appType int: Application type to request focus for.
ownershipCallback CarAppFocusManager.OnAppFocusOwnershipCallback: Ownership callback to request app focus for. Cannot be null.
Returns
int APP_FOCUS_REQUEST_FAILED or APP_FOCUS_REQUEST_SUCCEEDED
Throws
SecurityException if owner cannot be changed.
CarNotConnectedException if the connection to the car service has been lost.
